# [StoryG 실전] 과정 aiResult 도입기

같은 학생 답변이라도 과정 맥락이 다르면 리포트 톤과 포인트가 달라집니다. StoryG에서는 과정 생성/수정 시 `aiResult`를 입력받아 AI 프롬프트에 주입하는 구조를 도입했습니다.

## 🚨 문제

- AI 리포트가 문맥 없이 일반론적으로 생성되는 경우가 있음
- 과정 목표(예: 진로탐구, 디지털 시민교육)가 리포트에 충분히 반영되지 않음
- 운영자가 의도한 교육 초점이 출력에 반영되지 않는 불만 발생

## 🧪 재현

- 동일한 학생 응답 데이터
- 다른 과정 맥락
- 결과 리포트는 거의 비슷한 문장 구성

## 🔍 원인 분석

프롬프트에 과정 컨텍스트가 없었습니다.

- 입력 데이터: 학생 응답 중심
- 누락 데이터: 과정 목적/의도/핵심 역량

모델 입장에서는 "왜 이 활동을 했는지"를 알 수 없는 상태였습니다.

## 🛠️ 해결

`tbl_courses.ai_result`를 추가하고, 리포트 생성 시 프롬프트에 자동 주입했습니다.

```sql
ALTER TABLE tbl_courses ADD COLUMN ai_result LONGTEXT NULL;
```

요청 예시:

```json
{
  "title": "2026 진로탐구 과정",
  "aiResult": "이 과정은 학생들이 자신의 흥미와 적성을 발견하고 진로를 구체화하는 데 초점을 맞춥니다."
}
```

프롬프트 구성 예시:

```text
[과정 참고 정보]
이 과정은 학생들이 자신의 흥미와 적성을 발견하고 진로를 구체화하는 데 초점을 맞춥니다.

[학생 응답 요약]
...
```

## ✅ 검증

- 리포트 문맥 일관성 향상
- 과정 목표와 연결된 표현 증가
- 운영자 피드백에서 "과정 의도가 잘 보인다" 반응 증가

## 📌 교훈

1. AI 품질은 모델 교체보다 입력 컨텍스트 설계로 크게 개선됩니다.
2. 운영자가 프롬프트 컨텍스트를 조정할 수 있어야 실무 적용성이 높아집니다.
3. 선택 필드(`null` 허용)로 설계하면 기존 동작과의 호환성을 유지할 수 있습니다.

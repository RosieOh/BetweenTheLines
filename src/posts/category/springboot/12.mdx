# [Almighty 프로젝트] Submission 공통화 리팩터링

PPS, DIG, GCC, WFA를 운영하다 보니 제출 테이블이 모듈별로 분화되면서 공통 필드 중복이 커졌습니다. 결국 쿼리 복잡도와 유지보수 비용이 급증했고, `TBL_SUBMISSION` 공통화로 구조를 재정비했습니다.

처음에는 "모듈마다 요구사항이 다르니까 테이블도 다르면 되지"라고 생각했는데, 운영 지표를 하나로 모아보려는 순간 한계가 바로 드러났습니다.

## 🚨 문제

- 모듈별 제출 테이블에 `USER_ID`, `TASK_ID`, `CREATE_DT` 등 공통 필드 중복
- 공통 통계 쿼리 작성 시 UNION/조건 분기 증가
- 신규 모듈 추가 시 스키마/코드 중복 확산

## 🧪 재현

공통 통계를 만들 때 아래와 같은 문제가 반복되었습니다.

- 모듈마다 조인 경로 다름
- 동일한 지표를 모듈별로 각각 계산
- 운영 이슈 시 "전체 제출 관점" 추적 어려움

## 🔍 원인 분석

엔티티 경계가 도메인 기준이 아니라 구현 편의 기준으로 나뉘어 있었습니다.

- "제출"이라는 공통 도메인과
- "모듈별 특화 데이터"가
- 같은 레벨에서 분리 없이 설계됨

## 🛠️ 해결

구조를 다음처럼 재설계했습니다.

- 공통: `TBL_SUBMISSION`
- 확장: `TBL_SUBMISSION_PPS`, `TBL_SUBMISSION_DIG`, `TBL_SUBMISSION_GCC`, `TBL_SUBMISSION_WFA`
- 관계: `SUBMISSION_ID` 기준 One-to-One

장점:

1. 공통 조회 단순화
2. 모듈 확장 시 신규 확장 테이블만 추가
3. 데이터 일관성 향상

## 🔧 실전 코드: JPA 엔티티 분리 예시

```java
@Entity
@Table(name = "TBL_SUBMISSION")
public class Submission {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long submissionId;

    private Long userId;
    private Long taskId;
    @Enumerated(EnumType.STRING)
    private Category category; // PPS, DIG, GCC, WFA
}
```

```java
@Entity
@Table(name = "TBL_SUBMISSION_DIG")
public class SubmissionDig {
    @Id
    private Long submissionId;

    @MapsId
    @OneToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "SUBMISSION_ID")
    private Submission submission;

    private String filePath;
    @Lob
    private String analysisResult;
}
```

## ✅ 마이그레이션 절차 예시

1. 공통 테이블 생성
2. 모듈별 공통 필드를 `TBL_SUBMISSION`으로 백필
3. 특화 필드를 확장 테이블로 이관
4. 조회 쿼리/서비스 계층 전환
5. 검증 후 레거시 테이블 read-only 전환

## ✅ 검증

- 공통 제출 통계 쿼리 단순화
- 제출 기반 장애 분석 속도 개선
- 모듈별 데이터는 그대로 유지하면서 공통 흐름 표준화

## 📌 교훈

1. 공통 필드는 초기에 분리하지 않으면 나중에 기술 부채로 크게 돌아옵니다.
2. 도메인 공통성과 모듈 특수성은 테이블 계층으로 분리하는 것이 유리합니다.
3. 확장 가능성은 "새 기능 추가 비용"으로 측정해야 합니다.

## ♻️ 재사용 스키마 전략

- 공통 엔티티 1개 + 모듈 확장 엔티티 N개
- 공통 쿼리 우선, 특화 쿼리 후순위
- 마이그레이션은 "공통 먼저, 특화 다음" 순서로

## ✨ 마무리

리팩터링의 핵심은 코드가 아니라 모델입니다. 모델이 정리되면 쿼리와 서비스 코드는 자연스럽게 단순해집니다.  
Almighty에서는 이 구조 변경 이후, 신규 모듈 실험 속도가 확실히 빨라졌습니다.
